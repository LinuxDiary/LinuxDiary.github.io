# iptables

# 表和链

iptables 默认有 4 张表，每张表分别包含了不同的链，表和链的关系如下

        +-----------------+     +-----------------+     +-----------------+
        |                 |     |                 |     |                 |
        |      Table      | --- |      Chain      | --- |  Rules(Action)  |
        |                 |     |                 |     |                 |
        +-----------------+     +-----------------+     +-----------------+
                 |                       |                       |
        +-----------------+     +-----------------+     +-----------------+
        |                 |     |      INPUT      |     |      ACCEPT     |
        |      filter     | --- |     FORWARD     |     +-----------------+
        |                 |     |      OUTPUT     |              |
        +-----------------+     +-----------------+     +-----------------+
                 |                       |              |       DROP      |
        +-----------------+     +-----------------+     +-----------------+
        |                 |     |   PREROUTING    |              |
        |       nat       | --- |      INPUT      |     +-----------------+
        |                 |     |      OUTPUT     |     |      REJECT     |
        |                 |     |   POSTROUTING   |     +-----------------+
        +-----------------+     +-----------------+              |
                 |                       |              +-----------------+
        +-----------------+     +-----------------+     |       SNAT      |
        |                 |     |   PREROUTING    |     +-----------------+
        |                 |     |      INPUT      |              |
        |      mangle     | --- |     FORWARD     |     +-----------------+
        |                 |     |      OUTPUT     |     |       DNAT      |
        |                 |     |   POSTROUTING   |     +-----------------+
        +-----------------+     +-----------------+              |
                 |                       |              +-----------------+
        +-----------------+     +-----------------+     |    MASQUERADE   |
        |       raw       | --- |      OUTPUT     |     +-----------------+
        |                 |     |    PREROUTING   |              |
        +-----------------+     +-----------------+     +-----------------+
                                                        |     REDIRECT    |
                                                        +-----------------+
                                                                 |
                                                        +-----------------+
                                                        |       LOG       |
                                                        +-----------------+

数据表在 iptables 中的匹配流程和规则

图中反映了数据包的流向，同时也反映了同一个链中表的优先级，优先级由高到低 raw -> mangle -> nat -> filter

# 常用选项

选项 说明
-t 指定要操纵的表
-A 向规则链中添加条目
-D 从规则链中删除条目
-I 向规则链中插入条目
-R 替换规则链中的条目
-L 显示规则链中已有的条目
-F 清楚规则链中已有的条目
-Z 清空规则链中的数据包计算器和字节计数器
-X 删除用户自定义的空链
-N 创建新的用户自定义规则链
-P 定义规则链中的默认目标
-n 不显示服务名，只显示端口号
-p 指定要匹配的数据包协议类型
-s 指定要匹配的数据包源ip地址
-j 指定动作
-i 指定数据包进入本机的网络接口
-o 指定数据包离开本机的网络接口

# 命令基本语法

iptables -t 表名 <-A/I/D/R> 规则链名 [规则号] <-i/o 网卡名> -p 协议名 <-s 源IP/源子网> --sport 源端口 <-d 目标IP/目标子网> --dport 目标端口 -j 动作

使用 iptables 命令创建或更新的规则是临时生效，如果需要永久保存可以使用以下命令

$ sudo bash -c "iptables-save > /etc/sysconfig/iptables"

# 防火墙常用策略

# filter表

设置默认规则，默认允许数据包通过 INPUT FORWARD OUTPUT 链

iptables -P INPUT ACCEPT
iptables -P OUTPUT ACCEPT
iptables -P FORWARD ACCEPT

允许相关连接和已建立连接的数据包

iptables -I INPUT -m state --state RELATED,ESTABLISHED -j ACCEPT

允许 ping ，即允许 icmp 数据包

iptables -I INPUT -p icmp -j ACCEPT

允许本地环回地址的数据包

iptables -I INPUT -i lo -j ACCEPT

允许访问 22 端口

iptables -I INPUT -p tcp --dport 22 -j ACCEPT

拒绝所有数据包进入以及转发

iptables -A INPUT -j REJECT --reject-with icmp-host-prohibited
iptables -A FORWARD -j REJECT --reject-with icmp-host-prohibited

如下是 iptables 首次安装时所生成的规则

*filter
:INPUT ACCEPT [0:0]
:FORWARD ACCEPT [0:0]
:OUTPUT ACCEPT [0:0]
-A INPUT -m state --state RELATED,ESTABLISHED -j ACCEPT
-A INPUT -p icmp -j ACCEPT
-A INPUT -i lo -j ACCEPT
-A INPUT -p tcp -m state --state NEW -m tcp --dport 22 -j ACCEPT
-A INPUT -j REJECT --reject-with icmp-host-prohibited
-A FORWARD -j REJECT --reject-with icmp-host-prohibited
COMMIT

允许转发除了 icmp 以外的数据包

iptables -A FORWARD -p ! icmp -j ACCEPT

允许转发来自 192.168.1.0/24 网段中除了 192.168.1.10 的数据包

iptables -A FORWARD -s 192.168.1.10 -j REJECT
iptables -A FORWARD -s 192.168.1.0/24 -j ACCEPT

拒绝从 eth0 接口进入本机的源地址为 192.168.1.0/24 网段的数据包

iptables -A INPUT -i eth0 -s 192.168.1.0/24 -j REJECT

拒绝地址段 192.168.1.100-192.168.1.150 的数据包访问本机

iptables -I INPUT -p tcp -m iprange --src-range 192.168.1.100-192.168.1.150 -j REJECT

只允许 192.168.1.0/24 网段连接 22 端口

iptables -I INPUT -p tcp --dport 22 -s 192.168.1.0/24 -j ACCEPT
iptables -I INPUT -p tcp --dport 22 -j REJECT

允许访问本机的 3000-4000 端口

iptables -I INPUT -p tcp --dport 3000:4000 -j ACCEPT

允许访问本机的 22 443 80 8080 端口

iptables -I INPUT -p tcp -m multiport --dport 22,80,443,8080 -j ACCEPT

拒绝来自 MAC 地址为 00:E0:4C:38:86:68 的数据包

iptables -I INPUT -m mac --mac-source 00:E0:4C:38:86:68 -j REJECT

# nat表

私网地址通过 nat 地址转换连接互联网，假设私网接口 eth0 ， IP 192.168.1.1 ，公网接口 eth1 ， IP 202.103.24.68

iptables -t nat -A POSTROUTING -s 192.168.1.0/24 ! -d 192.168.1.0/24 -o eth1 -j MASQUERADE

公网 2122 端口映射到私网 22 端口

iptables -t nat -A PREROUTING -i eth1 -d 202.103.24.68 -p tcp --dport 2122 -j DNAT --to-destination 192.168.1.100:22

将访问本机的 80 端口重定向到 8888 端口，重定向的同时，需要在 filter 表中允许访问本机 8888 端口

iptables -t nat -A PREROUTING -i eth1 -p tcp --dport 80 -j REDIRECT --to-port 8888
iptables -I INPUT -p tcp --dport 8888 -j ACCEPT

SS 中转服务器规则示例, 中转服务器处于 NAT 环境中, 如果是非 NAT 环境, 就将内网地址换成服务器接口上的公网地址

iptables -t nat -A PREROUTING -d 10.40.55.131 -p tcp -m tcp --dport 4433 -j DNAT --to-destination 54.92.100.218:443
iptables -t nat -A PREROUTING -d 10.40.55.131 -p udp -m udp --dport 4433 -j DNAT --to-destination 54.92.100.218:443
iptables -t nat -A POSTROUTING -d 54.92.100.218 -p tcp -m tcp --dport 443 -j SNAT --to-source 10.40.55.131
iptables -t nat -A POSTROUTING -d 54.92.100.218 -p udp -m udp --dport 443 -j SNAT --to-source 10.40.55.131
