# lvm管理

  • 系统环境：CentOS 7
  • 软件版本：lvm2

# 新建

通常在安装操作系统时就创建 lvm ，这里是在已安装好操作系统且未配置过 lvm 的环境下创建新的 lvm

# 安装lvm2

yum install lvm2

# 创建物理卷

物理卷可以是一个分区，也可以是一整块磁盘

pvcreate /dev/sda2

# 创建卷组

一个卷组就是一个物理卷的集合，创建时可以指定多个已创建的物理卷，也可以向已存在的卷组中添加新的物理卷

vgcreate VGNAME /dev/sda2

# 创建逻辑卷

逻辑卷的创建就是从卷组中获取指定的容量，来创建一个虚拟的可格式化的磁盘分区，卷组就相当于一个由多个真实的物理卷组成的存储池，而逻辑卷不关心物理卷的大小和数量。

这里创建一个 20G 的逻辑卷

lvcreate -n LVNAME -L 20G VGNAME

# 使用逻辑卷

逻辑卷创建好之后，就和真实的物理磁盘分区一样，直接格式化就可以使用了，当然也可以添加到 fstab 文件中

mkfs.ext4 /dev/VGNAME/LVNAME
mount /dev/VGNAME/LVNAME /mnt

# 删除逻辑卷

删除之前，一定要注意逻辑卷分区中的数据。要删除逻辑卷，要遵循顺序 lv --> vg --> pv

lvremove /dev/VGNAME/LVNAME
vgremove VGNAME
pvremove /dev/sdb

# 动态扩展

  • 新增物理卷
pvcreate /dev/sdd
  • 把物理卷添加到卷组
vgextend VGNAME /dev/sdd
  • 扩展逻辑卷容量

扩展逻辑卷是热操作，可以直接在线执行，有以下两种扩展方式

增加10G到LVNAME卷中

lvextend -L +10G /dev/VGNAME/LVNAME

增加剩余所有容量到卷中

lvextend -l +100%FREE /dev/VGNAME/LVNAME　　
  • 更新文件系统容量

扩展容量后，只有更新之后才会生效

resize2fs /dev/VGNAME/LVNAME  #适用于ext2、3、4文件系统
xfs_growfs /dev/VGNAME/LVNAME  #适用于xfs文件系统

# 缩小逻辑卷

要执行逻辑卷缩小操作，必须先转移数据并取消挂载，且操作顺序必须是 umount --> resize2fs --> lvreduce ，假设总容量 20G ，要减少 10G 左右

  • 卸载逻辑卷
umount /dev/VGNAME/LVNAME
  • 重新定义逻辑卷大小
resize2fs /dev/VGNAME/LVNAME 10G
  • 减少逻辑卷容量

把逻辑卷较少 9G 容量，为防止数据丢失，减少的容量尽量少于实际计算所得容量

lvreduce -L -9G /dev/VGNAME/LVNAME

把逻辑卷减少到 9G ，缩小后的容量必须大于已经使用的空间

lvreduce -L 9G /dev/VGNAME/LVNAME
  • 重新挂载逻辑卷
mount /dev/VGNAME/LVNAME /mnt

# 缩小卷组

缩小卷组容量，退出物理卷（物理硬盘）。缩小前必须确认 vg 空闲容量大于要退出的物理卷容量 /dev/sdd ，否则可能造成数据丢失

vgreduce VGNAME /dev/sdd
pvremove /dev/sdd
