# NFS Server

CentOS7 下 NFS 服务中的 rpcbind 服务监听在 111 端口， NFS 主程序监听在 2049 端口。NFS 包安装时会自动创建 nfsnobody 用户和组， uid 和 gid 都是 65534 。

# 安装

安装 nfs-utils 的同时会自动安装 nfs-server 、 rpcbind 等包

yum -y install nfs-utils

# 端口

默认 NFS 在挂载时会随机分配 1024 以上端口，因此固定 NFS 服务的端口，便于配置防火墙规则。编辑 /etc/sysconfig/nfs 文件追加以下配置

MOUNTD_PORT=4001　　
STATD_PORT=4002
LOCKD_TCPPORT=4003
LOCKD_UDPPORT=4003
RQUOTAD_PORT=4004

# 共享目录

服务端和客户端均使用非 root 用户来管理共享目录(但挂载操作需要 root 权限)，要求两端使用具有相同 uid 的用户，例如 uid 都为 1111

# 创建NFS用户和共享目录

useradd -u 1111 nfsuser
passwd nfsuser
su - nfsuser
$ mkdir nfs_share_dir

TIP

如果挂载时发现 NFS 共享目录只能挂载为 nobody 用户，可以分别修改服务端和客户端 /etc/idmapd.conf 中的 Domain 选项为一致，例如 Domain = nfs.io

# 定义共享目录

配置要共享的目录，注意目录的用户组和权限。在共享参数里面设置匿名访问用户的本地用户 UID ，也就是客户端映射到服务端的用户 UID ，共享目录中的文件在两端都将以 UID=1111 和 GID=1111 为用户组保存。编辑 /etc/exports ，添加以下内容

/home/nfsuser/nfs_share_dir 10.20.20.0/24(rw,anonuid=1111,anongid=1111,all_squash)

配置完成之后，重载并查看目录列表

exportfs -rv

# 共享参数

选项 描述
rw 允许读写权限
ro 只读权限
sync 同步模式(Default)，所有数据在写入后可以请求
async 异步模式，数据在写入过程中可以写入
secure NFS通过1024以下的安全TCP/IP端口发送(Default)
insecure NFS可以通过所有端口发送
wdelay 如果多个用户要写入NFS目录，则归组写入 (Default)
no_wdelay 如果多个用户要写入NFS目录，则立即写入，当使用async时，无需此设置。
subtree_check 如果共享/usr/bin之类的子目录时，强制NFS检查父目录的权限 (Default)
no_subtree_check 与subtree_check对应
root_squash 将root用户映射为服务端NFS指定的匿名uid/gid
no_root_squash root用户具有根目录的完全管理访问权限。
all_squash Map all uids and gids to the anonymous user. 对公共目录访问时较有用。
no_all_squash 保留共享文件的UID和GID (Default)
anonuid=UID 指定匿名用户访问时映射机的用户uid
anongid=GID 指定匿名用户访问时映射机的用户gid

# 防火墙

按照先前配置的端口配置防火墙规则

iptables -I INPUT 5 -p tcp --dport 111 -j ACCEPT
iptables -I INPUT 5 -p udp --dport 111 -j ACCEPT
iptables -I INPUT 5 -p tcp --dport 2049 -j ACCEPT
iptables -I INPUT 5 -p udp --dport 2049 -j ACCEPT
iptables -I INPUT 5 -p tcp --dport 4001:4004 -j ACCEPT
iptables -I INPUT 5 -p udp --dport 4001:4004 -j ACCEPT
iptables-save >/etc/sysconfig/iptables

# 启动服务

systemctl start rpcbind.service
systemctl enable rpcbind.service
systemctl start nfs.service
systemctl enable nfs.service

# 挂载

uid 和 gid 与服务端保持一致，使用非 root 用户管理挂载目录。客户端同样需要事先安装 nfs-utils ，安装之后创建用于挂载的用户，这里与服务端用户保持相同的 uid

yum -y install nfs-utils
useradd -u 1111 nfsuser
passwd nfsuser

切换到挂载用户，创建挂载目录

su - nfsuser
mkdir nfs_share

与服务端一样，修改 idmapd.conf 中 Domain 选项，保持一致

vi /etc/idmapd.conf
Domain = nfs.io

挂载 nfs 类型需要 root 权限

mount -t nfs 10.20.20.136:/home/nfsuser/nfs_share_dir /home/mysql/nfs_share

挂载成功之后，可以使用 showmount -e IP/DOMAIN 查看 NFS 服务端目录输出列表，例如 showmount -e 10.20.20.136

# autofs

autofs 是一个自动按需挂载 NFS 的应用，需要在 nfs-utils 已安装的前提下运行

yum -y install nfs-utils autofs

# 挂载参数

vi /etc/auto.master
# 添加挂载点
/-    /etc/auto.mount

vi /etc/auto.mount
# 在挂载选项中设置挂载目录的用户 UID ，与服务端配置的 UID 保持一致，以保证非 root 用户权限挂载
/home/nfsuser/nfs_share    -fstype=nfs,rw,uid=1111,gid=1111    10.20.20.136:/home/nfsuser/nfs_share_dir

TIP

使用 autofs 时，本地挂载目录需使用完整的绝对路径，而不是包含软链接的路径，否则可能因无法挂载而报错 automount[7510]: do_mount_autofs_direct: failed to create mount directory ，并且本地挂载目录不能事先创建，这一点不同于手动 mount 挂载， autofs 启动后会自动创建本地目录然后挂载，例如这里需要定义的本地挂载目录 /home/nfsuser/nfs_share 但无须创建 nfs_share 目录。挂载成功时目录用户和组显示为 root ，对目录执行过动作之后会变为设定的 UID

与前面一样，修改 /etc/idmapd.conf 中 Domain 选项，保持一致

# /etc/idmapd.conf
Domain = nfs.io

# 重启服务

systemctl restart autofs.service

# 其他自动挂载方法

除了 autofs 以外，也可以直接在 /etc/fstab 里添加一条挂载记录，为了防止因无法挂载导致的不能进入系统，可以加入 _rnetdev 参数，表示无法挂载就跳过。

10.20.20.136:/home/nfsuser/nfs_share_dir    /home/mysql/nfs_share    nfs  defaults,_rnetdev  1  1
