# NTP Sever

在 CentOS 7 下分别使用 chrony 和 ntp 作为时间服务器

# NTPd

yum install ntp

# 配置文件

编辑配置文件 /etc/ntp.conf 更新以下内容

restrict default nomodify notrap nopeer noquery
restrict 127.0.0.1
# 系统若启用了ipv6，该项必须保留，否则会报错
restrict ::1

# 增加允许查询的网络范围
restrict 192.168.10.0 mask 255.255.255.0 nomodify notrap
restrict 10.20.20.0 mask 255.255.255.0 nomodify notrap

# 修改或保持默认的时间服务器列表
server 0.centos.pool.ntp.org iburst
server 1.centos.pool.ntp.org iburst
server 2.centos.pool.ntp.org iburst
server 3.centos.pool.ntp.org iburst

restrict 参数说明

参数 说明
ignore 拒绝连接到NTP服务器
nomodiy 忽略所有改变NTP服务器配置的报文，但可以查询配置信息
noquery 忽略所有mode字段为6或7的报文，客户端不能改变NTP服务器配置，也不能查询配置信息
notrap 不提供trap远程登录功能，trap服务是一种远程时间日志服务。
notrust 不作为同步的时钟源。
nopeer 提供时间服务，但不作为对等体。
kod 向不安全的访问者发送Kiss-Of-Death报文

# 启动服务

配置防火墙规则并启动服务

iptables -I INPUT -p udp --dport 123 -j ACCEPT
iptables-save > /etc/sysconfig/iptables
systemctl start ntpd.service
systemctl enable ntpd.service

# chrony

yum install chrony

# 配置文件

编辑配置文件 /etc/chrony.conf ，更新以下内容

# 使用默认的时间服务器或指定其他服务器
server 0.centos.pool.ntp.org iburst
server 1.centos.pool.ntp.org iburst
server 2.centos.pool.ntp.org iburst
server 3.centos.pool.ntp.org iburst

# 允许查询的客户端范围
allow 192.168.10.0/24
allow 10.20.20.0/24

iburst 选项表示在一个标准的轮询间隔内没有应答，客户端会发送 4 个包给 NTP 服务器，以便在客户端首次快速同步时间

# 启动服务

配置防火墙规则并启动服务

iptables -I INPUT -p udp --dport 123 -j ACCEPT
iptables-save > /etc/sysconfig/iptables
systemctl start chrony.service
systemctl enable chrony.service

# NTP客户端

使用NTP服务器软件

把上面配置中的 server 替换成内部 NTP 服务器即可，使用 prefer 参数将优先使用该服务器

server 10.20.20.2 prefer
server 10.20.20.3 iburst

使用ntpdate

安装 ntpdate ，并修改 /etc/ntp/step-tickers 。当然也可以使用默认的公共时间服务器，这里同样支持 iburst 或 prefer 等参数

# /etc/ntp/step-tickers

#0.centos.pool.ntp.org
10.20.20.2

启动服务

systemctl start ntpdate.service
systemctl enable ntpdate.service

TIP

需要注意的是， CentOS7 中的 ntpdate.service 服务只在系统启动时运行一次，不是定时运行。
