# 日志分类

# 查询日志

查询日志(general_log)记录了所有对 MySQL 数据库请求的信息，例如登陆数据库的信息、执行增删改查语句的信息、执行出错的信息等，无论这些请求的执行结果如何，都会一一记录。因此如果开启了查询日志就会产生大量的日志，非常消耗存储和 IO 资源。 MySQL 默认不启用查询日志，即 general_log=OFF 。查询日志会详细记录所有的请求，因此可以通过临时开启 general_log 来了解客户端的请求细节，例如分析 mysqldump 备份数据库的细节。

# 相关变量

SHOW GLOBAL VARIABLES LIKE '%general%';
+------------------+-------------------------------------------+
| Variable_name    | Value                                     |
+------------------+-------------------------------------------+
| general_log      | OFF                                       |
| general_log_file | /u01/mysql-5.6.39/data/mysql56_master.log |
+------------------+-------------------------------------------+

# 错误日志

错误日志(log_error)记录了 MySQL 启动、运行、关闭的过程(Note)，如果这些过程中出现了错误(Error)，就会在这个日志文件中进行记录，还会记录一些警告信息(Warning)。

# 相关变量

SHOW GLOBAL VARIABLES LIKE 'log_error';
+---------------+---------------------------------+
| Variable_name | Value                           |
+---------------+---------------------------------+
| log_error     | /u01/mysql-5.6.39/log/error.log |
+---------------+---------------------------------+

# 慢查询日志

慢查询日志(slow_query_log)可以记录可能查询很慢和没有使用索引的 SQL 语句。通过 long_query_time 设置一个阈值(毫秒级)，超过阈值的就记录慢查询；通过 log_queries_not_using_indexes 设置不使用索引就记录慢查询。在生产环境中，可能出现频繁的写慢查询日志，消耗系统 IO 和存储资源，从 MySQL5.6.5 版本以后还可以通过 log_throttle_queries_not_using_indexes 设置每分钟记录没有使用索引的慢查询的次数。

# 相关变量

SHOW GLOBAL VARIABLES LIKE '%long_query%';
+-----------------+----------+
| Variable_name   | Value    |
+-----------------+----------+
| long_query_time | 1.000000 |
+-----------------+----------+

SHOW GLOBAL VARIABLES LIKE '%slow_query%';
+---------------------+--------------------------------+
| Variable_name       | Value                          |
+---------------------+--------------------------------+
| slow_query_log      | ON                             |
| slow_query_log_file | /u01/mysql-5.6.39/log/slow.log |
+---------------------+--------------------------------+

SHOW GLOBAL VARIABLES LIKE '%log%not_using_indexes%';
+----------------------------------------+-------+
| Variable_name                          | Value |
+----------------------------------------+-------+
| log_queries_not_using_indexes          | ON    |
| log_throttle_queries_not_using_indexes | 5     |
+----------------------------------------+-------+

# 慢查询分析

查看查询结果(Rows)最多的 10 个查询：

mysqldumpslow -s ar -t 10 /u01/mysql-5.6.39/log/slow.log

查看锁的时间(Lock)最多的 10 个查询：

mysqldumpslow -s al -t 10 /u01/mysql-5.6.39/log/slow.log

查看查询时间最长的10个查询：

mysqldumpslow -s at -t 10 /u01/mysql-5.6.39/log/slow.log

过滤关键字查询，可以过滤SQL语句和索引相关的关键字，例如index、select、update等：

mysqldumpslow -g select -a /u01/mysql-5.6.39/log/slow.log

# 二进制日志

二进制日志(log_bin)用于记录对数据库更改的所有操作。 binlog 主要用于数据库备份恢复，主从复制。 MySQL 默认没有开启 log_bin ，可以在 my.cnf 中配置永久开启，即 log_bin = /u01/mysql-5.6.39/binlog/binlog

# 相关变量

SHOW GLOBAL VARIABLES LIKE '%log_bin%';
+---------------------------------+---------------------------------------+
| Variable_name                   | Value                                 |
+---------------------------------+---------------------------------------+
| log_bin                         | ON                                    |
| log_bin_basename                | /u01/mysql-5.6.39/binlog/binlog       |
| log_bin_index                   | /u01/mysql-5.6.39/binlog/binlog.index |
| log_bin_trust_function_creators | OFF                                   |
| log_bin_use_v1_row_events       | OFF                                   |
| sql_log_bin                     | ON                                    |
+---------------------------------+---------------------------------------+

# 查看/导出binlog

使用 mysqlbinlog 命令可以查看 binlog

mysqlbinlog mysqld-bin.000001 > test.sql
mysqlbinlog mysqld-bin.000035 -v     #查看row模式的binlog加-v参数

# 中继日志

中继日志是配置在 Slave 上的，用于主从复制。 Slave 从 Master 上获取 binlog ，存储到 relay-log 中，然后执行 relay-log 中的内容实现复制。

# 相关变量

SHOW VARIABLES LIKE '%relay%';
+---------------------------+-----------------------------------------+
| Variable_name             | Value                                   |
+---------------------------+-----------------------------------------+
| max_relay_log_size        | 0                                       |
| relay_log                 | /u01/mysql-5.6.39/binlog/relaylog       |
| relay_log_basename        | /u01/mysql-5.6.39/binlog/relaylog       |
| relay_log_index           | /u01/mysql-5.6.39/binlog/relay.index    |
| relay_log_info_file       | /u01/mysql-5.6.39/binlog/relay-log.info |
| relay_log_info_repository | FILE                                    |
| relay_log_purge           | ON                                      |
| relay_log_recovery        | OFF                                     |
| relay_log_space_limit     | 0                                       |
| sync_relay_log            | 10000                                   |
| sync_relay_log_info       | 10000                                   |
+---------------------------+-----------------------------------------+
