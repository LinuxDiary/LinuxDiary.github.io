# 重定向

以下是 example.com 重定向到 www.example.com 的重定向示例，也可以实用 302 临时重定向




 








server {
    listen       80;
    server_name  example.com;
    return       301 http://www.example.com$request_uri;
}

server {
    listen       80;
    server_name  www.example.com;
    ...
}

基于 https 的重定向示例，需要带上证书配置




 
 






 
 



server {
    listen       443 ssl;
    server_name  example.com;
    ssl_certificate      ssl/www.example.com.crt;
    ssl_certificate_key  ssl/www.example.com.key;
    return       301 https://www.example.com$request_uri;
}

server {
    listen       443 ssl;
    server_name  www.example.com;
    ssl_certificate      ssl/www.example.com.crt;
    ssl_certificate_key  ssl/www.example.com.key;
    ...
}

# 健康检查

使用 nginx_upstream_check_module 模块实现对 upstream 后端节点健康检查, 支持三中方式的结果输出 html cvs json

# 安装

git clone https://github.com/yaoweibin/nginx_upstream_check_module.git

选择合适的补丁版本, 对 Nginx 源码 patch

cd /home/nginx/u01/nginx/src/nginx-1.13.12
patch -p1 < /path/to/nginx_http_upstream_check_module/check_1.12.1+.patch

重新编译, 首先通过 nginx -V 获取编译参数, 追加 --add-module=/home/nginx/nginx_upstream_check_module-master 编译参数, 在 Nginx 源码目录执行编译

cd /home/nginx/u01/nginx/src/nginx-1.13.12
./configure --add-module=/home/nginx/nginx_upstream_check_module-master --prefix=/home/nginx/u01/nginx-1.13.12 --user=nginx --group=nginx --pid-path=/home/nginx/u01/nginx-1.13.12/run/nginx.pid --lock-path=/home/nginx/u01/nginx-1.13.12/run/nginx.lock --error-log-path=/home/nginx/u01/nginx-1.13.12/logs/error.log --http-log-path=/home/nginx/u01/nginx-1.13.12/logs/access.log --with-http_ssl_module --with-http_stub_status_module --with-http_gzip_static_module --with-http_stub_status_module --with-pcre --with-file-aio --with-http_realip_module --with-http_v2_module --with-stream

然后只执行 make 命令, 会在 objs 目录下生成编译好的 nginx 可执行文件, 停止 nginx 服务, 备份原 nginx 程序, 将新 nginx 可执行文件拷贝至 nginx 目录下的 sbin 目录中

make
nginx -s quit
mv ~/home/u01/nginx/sbin/nginx ~/home/u01/nginx/sbin/nginx.bak
cp objs/nginx ~/home/u01/nginx/sbin/

最后修改 nginx.conf 配置文件, 添加健康检查选项

...
upstream tomcat {
    server 192.168.5.5:8080 weight=5;
    server 192.168.5.6:8080 weight=2;
    ip_hash;
    check interval=5000 rise=1 fall=3 timeout=4000;
}
...

server {
    listen       8080 so_keepalive=30::3;
    server_name  localhost;

...
    location /status {
            check_status;

            access_log   off;
            allow all;
       #     deny all;
    }
...
}

重启 nginx 后, 可通过 http://IP:8080/status 查看节点状态列表, 支持三种格式

  • http://IP:8080/status?format=html
  • http://IP:8080/status?format=csv
  • http://IP:8080/status?format=json

参考链接：http://nginx.org/en/docs/http/converting_rewrite_rules.html (opens new window)
