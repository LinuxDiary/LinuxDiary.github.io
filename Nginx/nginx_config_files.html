# 配置文件

在编译 Nginx 时如果没有特别指定配置文件的路径，默认 Nginx 的配置文件目录位于安装目录下的 conf 目录，如 /home/www/u01/nginx/conf 。Nginx配置文件主要由4个部分组成：全局配置、events {} 、http {} 、mail {} 每个 {} 为一个块。

# 主配置文件

## nginx.conf

# nginx运行用户
user www;
# 启动进程,通常设置成和cpu的数量相等
worker_processes  1;

# 全局错误日志及PID文件，一般在编译时指定了路径，这里可以不写
# error_log  logs/error.log;
# error_log  logs/error.log  notice;
# error_log  logs/error.log  info;

# pid        run/nginx.pid;

# 工作模式及连接数上限
events {
    # epoll是多路复用IO(I/O Multiplexing)中的一种方式,
    # 仅用于linux2.6以上内核,可以大大提高nginx的性能
    use   epoll;

    # 单个后台worker_process进程的最大并发连接数，默认是1024   
    worker_connections  4096;
}

charset UTF-8;

http {
    # 设定mime类型,类型由mime.type文件定义
    include    mime.types;
    default_type  application/octet-stream;
    # 设定日志格式
    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    # 使用日志格式定义访问日志路径
    access_log  logs/access.log  main;

    # sendfile 指令指定 nginx 是否调用 sendfile 函数（zero copy 方式）来输出文件，
    # 对于普通应用，必须设为 on,
    # 如果用来进行下载等应用磁盘IO重负载应用，可设置为 off，
    # 以平衡磁盘与网络I/O处理速度，降低系统的uptime.
    sendfile     on;
    # tcp_nopush     on;

    # 连接超时时间
    # keepalive_timeout  0;
    keepalive_timeout  65;
    tcp_nodelay     on;

    # 开启gzip压缩
    gzip  on;
    gzip_disable "MSIE [1-6].";

    # 设定请求缓冲
    client_header_buffer_size    128k;
    large_client_header_buffers  4 128k;

    # 定义上游服务池
    upstream php {
        server 127.0.0.1:9000;
    }

    # 虚拟主机配置
    server {
        # 侦听80端口
        listen    80;
        # 定义使用 www.example.com访问
        server_name  www.example.com;

        # 网站根目录
        root html;

        # 本虚拟主机的访问日志
        access_log  logs/nginx.access.log  main;

        # 默认请求
        location / {
            # 定义首页索引文件的名称
            index index.php index.html index.htm;   
        }

        # 定义错误提示页面
        error_page   500 502 503 504 /50x.html;
        location = /50x.html {
        }

        # 静态文件，nginx自己处理
        location ~ ^/(images|javascript|js|css|flash|media|static)/ {

            # 过期30天，静态文件不怎么更新，过期可以设大一点，
            # 如果频繁更新，则可以设置得小一点。
            expires 30d;
        }

        # PHP 脚本请求全部转发到上游FastCGI处理，上游可以是本机或者远程机器
        location ~ \.php$ {
            fastcgi_index index.php;
            include fastcgi.conf;
            fastcgi_pass php;
        }

        # 禁止访问 .htxxx 文件
            location ~ /\.ht {
            deny all;
        }

    }
}

TIP

Nginx 配置文件中所使用的相对路径(如网站根目录 root html; )都是以 Nginx 根目录为基准，也就是编译时指定的参数 --prefix= ，如 --prefix=/home/www/u01/nginx-1.13.12

# 虚拟主机

一个 server {} 块为一个虚拟主机，置于 http {} 中，通常将每个虚拟机的配置写到单独的文件中，然后在主配置文件 http {} 中使用 include 指令包含进来，如：

## nginx.conf

...
http {
  ...
  include site/*.conf;
}

server {} 中由 listen 和 server_name 指令相结合来定义一个虚拟主机，一般有三种定义方式：

基于IP

server {
    listen 80;
    server_name 192.168.2.22;
    location / {
            root html/server1;
            index index.html index.htm index.php;
    }
}
server {
    listen 80;
    server_name 192.168.2.23;
    location / {
            root html/server2;
            index index.html index.htm index.php;
    }
}

基于端口

server {
    listen 80;
    server_name www.example.com;
    location / {
            root html/server1;
            index index.html index.htm index.php;
    }
}
server {
    listen 81;
    server_name www.example.com;
    location / {
            root html/server2;
            index index.html index.htm index.php;
    }
}

基于主机名(域名)

server {
    listen 80;
    server_name doc.example.com;
    location / {
            root html/server1;
            index index.html index.htm index.php;
    }
}
server {
    listen 80;
    server_name bbs.example.com;
    location / {
            root html/server2;
            index index.html index.htm index.php;
    }
}

在基于主机名定义虚拟主机时，server_name 可以指定多个主机名，且支持通配符和正则表达式

通配符

server_name   .example.com   www.example.*;

正则表达式

使用正则表达式时，在主机名前面加 ~ 符号

server_name   www.example.com   ~^www\d+\.example\.com$;

# 虚拟主机基本匹配规则

Nginx 首先判断请求的 IP 和 Port ，然后在匹配的 listen 指令中选择一个能匹配请求的主机名 server_name ，如果没有匹配到主机名，则使用第一个匹配 IP 和 Port 的虚拟主机(即默认虚拟主机)进行响应，也可以使用 default_server 指定默认虚拟主机。

server {
    listen      192.168.100.1:80;
    server_name example.org www.example.org;
    ...
}

server {
    listen      192.168.100.1:80 default_server;
    server_name example.net www.example.net;
    ...
}

server {
    listen      192.168.100.1:80;
    server_name example.com www.example.com;
    ...
}

server {
    listen      192.168.100.2:80;
    server_name example.com www.example.com;
    ...
}
  • 从 192.168.100.1:80 上请求 www.example.com ，能匹配虚拟主机3，使用虚拟主机3的配置进行响应；
  • 从 192.168.100.1:80 上请求 www.longshuai.com 时，无法匹配任何虚拟主机，使用默认虚拟主机2的配置进行响应。如果将 listen 的属性 default_server 去掉，则使用虚拟主机1进行响应；
  • 从 192.168.100.2:80 上请求 www.example.com 时，使用虚拟主机4进行响应；
  • 从 192.168.100.2:80 上请求 www.example.org 时，没有相匹配的主机名，使用默认虚拟主机响应，即虚拟主机4。
