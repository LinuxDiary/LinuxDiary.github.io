# 并发连接数和客户端数

# 浏览器作为客户端建立连接

Nginx 作为 http 服务器时

max_clients = worker_processes * worker_connections/2

Nginx 作为反向代理时

max_clients = worker_processes * worker_connections/4

一个浏览器客户端在与 Nginx 建立连接时，会一次性建立 2 个连接，因此这里分别除 2 和 4

# 建立一般连接

Nginx 作为 http 服务器时

max_clients = worker_processes * worker_connections

Nginx 作为反向代理时

max_clients = worker_processes * worker_connections/2

TIP

max_clients 是最大客户端数量，而不是最大连接数，最大连接数一直都是 worker_processes * worker_connections

# 系统打开文件限制

Nginx 最大连接数首先受 worker_processes * worker_connections 限制，更主要的是受用户或进程可打开文件数量的限制，而用户或进程可打开文件数量限制不能超过系统所支持的最大可打开文件数的限制 file-max 。

$ ulimit -a | grep 'open files'    # 或者ulimit -n
open files                      (-n) 65535

$ cat /proc/sys/fs/file-max
98238

$ free
              total        used        free      shared  buff/cache   available
Mem:        1016252      227376      362208        9700      426668      576200

其中 file-max 默认约为内存总量的 10% ，系统会根据服务器的配置自动计算该值，最好不要修改为超过 10% 。运行 Nginx 服务的用户所能打开文件的最大数必须小于 file-max ，可根据 file-max 值适当调整用户的打开文件数，在 /etc/security/limits.d/20-nproc.conf 中进行调整。因此可根据 file-max 和 ulimit -n 适当调整 worker_connections
