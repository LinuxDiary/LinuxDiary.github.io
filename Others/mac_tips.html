# Mac技巧

# 终端颜色

ls 命令和 grep 命令高亮设置 编辑 ~/.bash_profile ，添加如下内容，然后 source ~/.bash_profile 生效

export CLICOLOR=1
export GREP_OPTIONS='--color=always'

vim 语法高亮 编辑 ~/.vimrc ，添加如下内容，然后 source ~/.vimrc 生效

syntax on

# 挂载NTFS

Mac 默认只能以只读模式挂载 NTFS 。通过 ntfs-3g 软件重新挂载 NTFS ，实现可读写

# 安装ntfs-3g

brew cast install osxfuse
brew install ntfs-3g

# 重新挂载

首先创建空目录，用于挂载 NTFS 分区

mkdir -p /User/ghost/Desktop/disk{1,2}

插上硬盘后 Mac 会自动以只读方式挂载，手动卸载磁盘后，按以下方式重新挂载(需要root权限)，即可正常读写 NTFS 磁盘

sudo umount /dev/disk2s1
sudo umount /dev/disk2s2
sudo mount -t ntfs -o rw,nobrowse /dev/disk2s1 /Users/ghost/Desktop/disk1
sudo mount -t ntfs -o rw,nobrowse /dev/disk2s2 /Users/ghost/Desktop/disk2

如果上一次使用硬盘时未正常弹出，再接入到 Mac 上，可能出现无法挂载的错误，例如:

sudo mount -w -t ntfs -o rw,auto,nobrowse /dev/disk2s1 /Users/ghost/Desktop/disk1
mount_ntfs: /dev/disk2s1 on /Users/ghost/Desktop/disk1: Read-only file system

此时需要修复磁盘，可以连到 Windows 系统中右键选择磁盘修复，或者在 Mac 上使用 ntfs-3g 命令挂载一次，这个过程会自动修复磁盘问题，例如

sudo ntfs-3g /dev/disk2s2 /Users/ghost/Desktop/disk2 -olocal -oallow_other
The disk contains an unclean file system (0, 0).
The file system wasnt safely closed on Windows. Fixing.

退出磁盘后，重新挂载。

      File "/private/var/folders/y1/sztsyy796jlb9b7jnk303gj80000gn/T/pip-install-q9gylwmm/mysqlclient/setup_posix.py", line 53, in get_config
        libs = mysql_config("libs_r")
      File "/private/var/folders/y1/sztsyy796jlb9b7jnk303gj80000gn/T/pip-install-q9gylwmm/mysqlclient/setup_posix.py", line 28, in mysql_config

    OSError: mysql_config not found

brew install mysql-connector-c
vi `which mysql_config`

libs="-L$pkglibdir"
libs="$libs -l "

修改成

libs="-L$pkglibdir"
libs="$libs -lmysqlclient -lssl -lcrypto"

重新安装 mysqlclient

pip install mysqlclient

TIP

python 在连接 MySQL 时需要使用 MySQL 的库文件，在安装 MySQL 时库文件可能位于 /usr/local/mysql/lib/ 或其他相似目录下，并不在 /usr/local/lib/ 下，此时需要做一个软连接让程序能够找到 MySQL 库文件的位置，例如 ln -s /usr/local/mysql/lib/libmysqlclient.18.dylib /usr/local/lib/libmysqlclient.18.dylib
