# 服务器规划

# CPU

  • 在业务上线初期，或者并发量较小的情况下，以双核至强或四核酷睿处理器作为起步参考
  • 常见的 MySQL Oracle WEB 应用，且并发在 1000 以下，以 四核至强*2 作为起步参考

# 内存

  • 6000 并发 --> 16G 起步
  • Tomcat --> 8G 参考
  • MySQL --> 16G 起步
  • Postfix/Exchange --> 2G 参考
  • 缓存(redis, memcached) --> 根据实际情况越大越好
  • NFS/Samba --> 1G 参考

# 存储

  • 优先选择 SSD ，否则尽可能选择转速高的 HD

# Raid

  • 缓存 --> Raid 0
  • Nginx/Apache/Tomcat --> Raid 1 / Raid 5
  • MySQL/Oracle --> Raid 10 / Raid 5
  • 普通应用 --> Raid 5

# 网络

  • 千兆*2 起步，高并发或存储相关业务使用 万兆*2 起步
