# JDK

# 安装

sudo tar xf jdk-8u181-linux-x64.tar.gz
sudo mv jdk1.8.0_181 /usr/local/

jdk 下载地址：http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html (opens new window)

# 环境变量

sudo vi /etc/profile.d/jdk.sh 
export JAVA_HOME=/usr/local/jdk1.8.0_181
export PATH=$JAVA_HOME/bin:$PATH

source /etc/profile.d/jdk.sh 
java -version

# 安装Tomcat

jdk 与 Tomcat 之间的版本依赖关系：https://tomcat.apache.org/whichversion.html (opens new window)

这里安装 Tomcat 8.5

mkdir -p u01
wget http://mirrors.tuna.tsinghua.edu.cn/apache/tomcat/tomcat-8/v8.5.34/bin/apache-tomcat-8.5.34.tar.gz
tar xf apache-tomcat-8.5.34.tar.gz
mv apache-tomcat-8.5.34 u01/

# Tomcat优化

# 线程池优化

配置项为 Connector

<Connector port="8080" protocol="org.apache.coyote.http11.Http11AprProtocol"
               maxConnections="2000"
               maxThreads="500"
               acceptCount="500"
               minSpareThreads="100"
               compression="on"
               compressionMinSize="2048"
               connectionTimeout="20000"
               URIEncoding="UTF-8"
               redirectPort="8443" />
  • maxConnections : 最大连接数限制，一般生产环境设置成 2000-3000
  • maxThreads : Tomcat 可生成的最大线程数，受系统 open files 限制，一般设置 500-700 默认为 200
  • acceptCount : 待 Tomcat 线程池满了以后，可接收的最大队列长度，一般设置成与 maxThreads 一样，如果 acceptCount 也满了，那么 Tomcat 会拒绝连接，提示 connection refused 默认为 100
  • minSpareThreads : 最小活动线程数， Tomcat 会始终保持这些线程在运行状态，默认为 10
  • compression : 是否开启 Gzip 压缩
  • compressionMinSize : 启用压缩时，最小的文件输出大小，默认为 2048 ，单位是 byte
  • connectionTimeout : 连接超时时间，默认为 20000ms

# JVM优化

vi catalina.sh
JAVA_OPTS="-server -Xms1024m -Xmx1024m -XX:MetaspaceSize=256m -XX:MaxMetaspaceSize=256m -XX:MaxNewSize=32m"
  • -server : 以 server 模式启动
  • -Xms : JVM 最小可用内存，一般配置为与 -Xmx 一样，避免多余的内存调度
  • -Xmx : JVM 最大可用内存，一般可设置成系统可用内存的 80%
  • -XX:MetaspaceSize : 元空间初始大小，一般设置 256m ，与 MaxMetaspaceSize 设成一致
  • -XX:MaxMetaspaceSize : 最大元空间大小，一般设置 256m
  • -XX:MaxNewSize : 最大新生代大小，一般设置成与 NewSize 一致

# Tomcat IO优化

Tomcat 支持三种 IO 模式

  • BIO : 阻塞处理 Request
  • NIO : 异步 IO ，非阻塞处理 Request
  • APR : 原生 C 语言编写的的非阻塞 IO

其中 NIO 是 Tomcat 默认的 IO 模式， APR 是性能最稳定，效率最高的 IO 模式，在 Tomcat 中使用 APR 的方法如下：

yum -y install apr apr-devel apr-util
cd tomcat/bin/
tar xf tomcat-native.tar.gz
cd tomcat-native-1.2.17-src
./configure (--with-apr=)
make 
sudo make install
vi catalina.sh
LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/apr/lib
export LD_LIBRARY_PATH
vi server.xml
protocol="org.apache.coyote.http11.Http11AprProtocol"

使用 org.apache.coyote.http11.Http11AprProtocol 替代 Tomcat 默认的 HTTP/1.1 ，表示启用 APR 模式，设置完成之后，重启 Tomcat 生效

# Tomcat完整配置文件

<?xml version="1.0" encoding="UTF-8"?>

<Server port="8005" shutdown="SHUTDOWN">
  <Listener className="org.apache.catalina.startup.VersionLoggerListener" />
  <Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="on" />
  <Listener className="org.apache.catalina.core.JreMemoryLeakPreventionListener" />
  <Listener className="org.apache.catalina.mbeans.GlobalResourcesLifecycleListener" />
  <Listener className="org.apache.catalina.core.ThreadLocalLeakPreventionListener" />

  <GlobalNamingResources>
    <Resource name="UserDatabase" auth="Container"
              type="org.apache.catalina.UserDatabase"
              description="User database that can be updated and saved"
              factory="org.apache.catalina.users.MemoryUserDatabaseFactory"
              pathname="conf/tomcat-users.xml" />
  </GlobalNamingResources>

  <Service name="Catalina">
    <Connector port="8080" protocol="org.apache.coyote.http11.Http11AprProtocol"
         maxConnections="2000"
         maxThreads="500"
         acceptCount="500"
         minSpareThreads="100"
         compression="on"
         compressionMinSize="2048"
         connectionTimeout="20000"
         redirectPort="8443" />
 
    <Connector port="8009" protocol="AJP/1.3" redirectPort="8443" />

    <Engine name="Catalina" defaultHost="localhost">

      <Realm className="org.apache.catalina.realm.LockOutRealm">
        <Realm className="org.apache.catalina.realm.UserDatabaseRealm"
               resourceName="UserDatabase"/>
      </Realm>

      <Host name="localhost"  appBase="webapps"
        <!-- <context path="/xuexi" docBase=/www/a/xuexi /> -->
        <Valve className="org.apache.catalina.valves.AccessLogValve" directory="logs"
               prefix="localhost_access_log" suffix=".txt"
               pattern="%h %l %u %t &quot;%r&quot; %s %b" />

      </Host>
    </Engine>
  </Service>
</Server>

如果要使用 80 端口，需要以 root 权限启动 Tomcat 。其中 appBase 是所有 web 应用的根目录， docBase 是每个 web 应用的程序根目录，类似于 nginx 的 root , path 是与之对应的 URI 。
