# Django项目部署

采用 uwsgi + nginx 部署 Django 项目，uwsgi 处理动态请求，由 nginx 接收请求并分发。

# settings

修改如下选项

DEBUG = False
ALLOWED_HOSTS = ['192.168.1.2']
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'static')
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

# 收集静态文件

这个步骤会根据前面配置的 settings.py 的选项，将所需的静态文件都拷贝到对应的目录中，这里是 STATIC_ROOT 和 MEDIA_ROOT ，进入到虚拟环境后，在项目根目录下执行如下命令

workon holaenv
python manage.py collectstatic

# uwsgi

# 安装并测试

切换到项目虚拟环境，安装 uwsgi

workon holaenv
pip install uwsgi

安装完成后，测试项目是否能正常运行，在项目根目录下运行

uwsgi --http :8000 --module hola.wsgi

其中 hola.wsgi 表示项目根目录中 hola 目录下的 wsgi.py 。访问 192.168.1.2:8000 如果能返回页面，表示运行成功，且此时静态文件应该是加载失败

在项目根目录下创建 uwsgi 目录，然后在其中创建配置文件 uwsgi.ini ，并填入以下内容

[uwsgi]
chdir = /home/nginx/u01/hola
module = hola.wsgi:application
master = True
processes = 10
pidfile = /home/nginx/u01/hola/uwsgi/uwsgi.pid
socket = 127.0.0.1:8000
vacuum = True
virtualenv = /home/nginx/.local/holaenv
daemonize = /home/nginx/u01/hola/uwsgi/hola.log

使用 uwsgi -i uwsgi/uwsgi.ini & 启动项目, 使用 uwsgi --stop uwsgi/uwsgi.pid 来关闭 uwsgi 进程

# Nginx

Nginx 负责接收请求，然后交给指定的程序处理，以下是配置示例

worker_processes  auto;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;

    upstream hola {
	      server 127.0.0.1:8000;
    }
    server {
        listen       8888;
        server_name  localhost;
        location / {
            uwsgi_pass hola;
            include uwsgi_params;
        }
        location /static/ {
            alias   /home/nginx/u01/hola/static/;
        }
        location /media/ {
            alias   /home/nginx/u01/hola/media/;
        }
    }
}

配置好以后，启动 nginx 直接访问 192.168.1.2:8888 即可访问
